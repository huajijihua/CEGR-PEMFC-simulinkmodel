# 模型模块核查

## BenchAirSupplySimplified

### 模块定位

`BenchAirSupplySimplified` 是气路最外侧的新鲜供气模块。它不计算台架供气流量，也不计算压降；它只把脚本已经整理好的台架湿空气组分、入口温度和入口压力转换成模型统一气体向量 `fresh`，供下游 `EGRMixerSimplified` 使用。

### 输入

- `boundary`：来自 `CaseBoundaryParam_simplified` 常量块，当前模块实际读取：
  - `boundary(2)`：阴极入口绝压，单位 `kPa`。
  - `boundary(3)`：阴极入口温度，单位 `degC`。
  - `boundary(5)`：台架新鲜供气 O2 质量分数。
  - `boundary(6)`：台架新鲜供气 N2 质量分数。
  - `boundary(7)`：台架新鲜供气水蒸气质量分数。

这些质量分数由初始化脚本根据 `combined_noegr_cegr_fit_points.csv` 中的 `bench_supply_p_actual_kPa`、`bench_supply_T_C`、`bench_supply_RH_pct` 换算得到；无独立台架供气状态时，脚本按入堆 p/T/RH 兜底。

### 计算内容

1. 直接读取 `boundary(5:7)`，不对负质量分数做静默截断。
2. 使用 `assert` 检查 O2、N2、水蒸气质量分数非负，且三者总和大于 0；如果不满足，说明输入数据或脚本映射存在错误，应中断仿真。
3. 对 O2、N2、水蒸气质量分数重新归一化，避免上游四舍五入或 CSV 精度造成三者和轻微偏离 1。
4. 生成不含液态水的新鲜供气向量。液态水项固定为 0，`wetFlag` 固定为 0。

### 输出

`fresh` 是 7x1 统一气体向量：

```text
fresh = [
  O2_mass_fraction
  N2_mass_fraction
  H2O_vapor_mass_fraction
  H2O_liquid_mass
  T_C
  p_kPa
  wetFlag
]
```

注意：`fresh(1:3)` 在该模块输出时是归一化质量分数，不是质量流量。实际入堆总质量流量由 `EGRMixerSimplified` 使用 `boundary(4)` 施加。

### 连线关系

- 输入 `boundary` 来自 `CaseBoundaryParam`。
- 输出 `fresh` 同时连接到：
  - `EGRMixerSimplified.fresh`
  - `fresh_node` 工作区记录块

### 核查结论

- 当前接口与脚本参数定义一致。
- 当前模块只负责供气组分和入口 p/T 的向量化，不负责流量控制；这与“阴极入口质量流量由实验入堆流量直接输入”的当前建模口径一致。
- 原 `physical` 输入在该模块未被使用；经核查其他模块均实际使用 `physical` 后，已删除 `BenchAirSupplySimplified` 的该冗余输入端口。
- 已在 MATLAB Function 内补充中文注释，说明输入索引、计算逻辑和输出向量约定；当前函数签名为 `function fresh = f(boundary)`。
- 已取消负质量分数的静默 `max(x,0)` 截断。实验边界错误应由初始化脚本和模块断言暴露，而不是在模型内部被自动修正。

### 验证记录

使用 `cegr0608_002` 工况做 1 秒冒烟仿真，`fresh_node` 输出为 7x1x51：

```text
fresh_sum = 1
T_C = 51.72
p_kPa = 141.085
wetFlag = 0
```

## EGRMixerSimplified

### 模块定位

`EGRMixerSimplified` 位于 `BenchAirSupplySimplified` 和 `BenchInletConditioner` 之间。它把新鲜供气和上一时间步延迟回来的 EGR 气体混合，并把混合后总质量流量整理为实验表给定的入堆目标流量。

该模块不直接读取 `boundary(11)` 的 EGR 比例。EGR 比例先在 `EGRSplitterSimplified` 中用于电堆出口分流，经过 `WaterSeparatorSimplified` 和 `EGRFeedbackDelay` 后形成实际回流向量 `egr`，再进入本模块混合。

### 输入

- `fresh`：来自 `BenchAirSupplySimplified` 的 7x1 新鲜供气向量。当前模块把 `fresh(1:4)` 作为组分质量分数使用。
- `egr`：来自 `EGRFeedbackDelay` 的 7x1 回流向量。当前模块把 `egr(1:4)` 作为回流质量流量使用。
- `boundary`：来自 `CaseBoundaryParam_simplified` 常量块，当前模块实际读取：
  - `boundary(4)`：阴极入堆目标总质量流量，单位 `kg/s`，由 `stack_in_flow_meter_SLPM` 换算得到。

### 计算内容

1. 检查 `boundary(4)` 为正，新鲜气和回流气前四项非负，新鲜气温度/压力在合理范围内。
2. 将 `fresh(1:4)` 归一化为新鲜气组分比例 `freshFrac`。
3. 计算可用 EGR 气相质量流量 `availableEgrGas = sum(egr(1:3))`。液态水不用于决定回流气体替代量，只作为组分携带。
4. 使用 `egrToMix = min(availableEgrGas, 0.95*mTarget)` 限制 EGR 最多替代 95% 的入堆目标流量，保留至少 5% 新鲜气补充。这是当前简化模型的物理/数值边界，不是输入数据纠错。
5. 新鲜气补充量为 `freshMakeup = mTarget - egrToMix`。
6. 如果实际混入 EGR，则把 `egr(1:4)` 归一化为回流组分比例 `egrFrac`；否则回流组分按 0 处理。
7. 按质量流量加权混合 O2/N2/H2O 气相和液相，并按质量流量加权混合温度。
8. 压力沿用新鲜气压力；后续 `BenchInletConditioner` 会用实测入堆压力再次覆盖。
9. 根据液态水质量流量设置 `wetFlag`。

### 输出

`mix` 是 7x1 混合气向量：

```text
mix = [
  O2_mass_flow
  N2_mass_flow
  H2O_vapor_mass_flow
  H2O_liquid_mass_flow
  mixed_T_C
  fresh_supply_p_kPa
  wetFlag
]
```

`mix(1:4)` 是质量流量，不是质量分数。正常情况下 `sum(mix(1:4))` 应等于 `boundary(4)`。

### 连线关系

- 输入 `fresh` 来自 `BenchAirSupplySimplified.fresh`。
- 输入 `egr` 来自 `EGRFeedbackDelay.y1`，即 `WaterSeparatorSimplified.gasOut` 的一步延迟。
- 输入 `boundary` 来自 `CaseBoundaryParam`。
- 输出 `mix` 同时连接到：
  - `BenchInletConditioner.mix`
  - `mixer_node` 工作区记录块

### 核查结论

- 当前接口与脚本参数定义一致：目标入堆流量来自 `stack_in_flow_meter_SLPM` 换算后的 `boundary(4)`。
- 已取消 `boundary(4)`、负回流量、负新鲜气组分的静默 `max` 保护，改为断言暴露错误。
- 初始化脚本已同步取消 `egr_fraction_model` 的静默 `min/max` 夹取，并对 EGR 比例、入堆流量、新鲜供气流量增加硬校验。
- 保留 `min(availableEgrGas, 0.95*mTarget)`，因为它表达“当前简化模型至少保留 5% 新鲜气补充”的混合边界。
- 该模块只负责新鲜气和回流气的质量混合，不计算 EGR 分流比例本身，也不最终决定入堆压力。

### 验证记录

## BenchInletConditioner

### 模块定位

`BenchInletConditioner` 位于 `EGRMixerSimplified` 和 `PEMFCStackCore` 之间。它代表台架测得的电堆阴极入口边界：混合器负责给出进入电堆前的质量流量和组分，本模块负责把入口温度、入口压力强制更新为实测入堆边界，并处理入口温压下的水蒸气饱和限制。

### 输入

- `mix`：来自 `EGRMixerSimplified` 的 7x1 混合气向量：

```text
mix = [
  O2_mass_flow
  N2_mass_flow
  H2O_vapor_mass_flow
  H2O_liquid_mass_flow
  T_C
  p_kPa
  wetFlag
]
```

- `physical`：来自 `PhysicalParam_simplified` 常量块，当前模块实际读取：
  - `physical(3)`：O2 摩尔质量，单位 `kg/mol`。
  - `physical(4)`：N2 摩尔质量，单位 `kg/mol`。
  - `physical(5)`：H2O 摩尔质量，单位 `kg/mol`。
- `boundary`：来自 `CaseBoundaryParam_simplified` 常量块，当前模块实际读取：
  - `boundary(2)`：阴极入口绝压，单位 `kPa`。
  - `boundary(3)`：阴极入口温度，单位 `degC`。

### 计算内容

1. 检查物性摩尔质量为正、入口压力为正、入口温度在饱和水蒸气压公式适用范围 `-40..120 degC` 内、混合器输出的前四个质量流量非负。
2. 将 `mix` 复制为 `caIn`，保留混合器输出的 O2/N2/H2O 气相和液相质量流量。
3. 用实测入堆边界覆盖 `caIn(5)` 和 `caIn(6)`，即入口温度和入口绝压。
4. 根据入口温度计算饱和水蒸气压 `pSat`，并用 `min(pSat, 0.98*p)` 限制水蒸气分压不能接近或超过总压。这里是物理上限，不是输入纠错。
5. 根据干气摩尔流量和入口温压计算可携带的饱和水蒸气质量流量 `mVaporSat`。
6. 如果 `caIn(3)` 超过 `mVaporSat`，将超出的水蒸气转入 `caIn(4)` 液态水。
7. 根据液态水质量流量设置 `wetFlag`。

### 输出

`caIn` 是 7x1 阴极入口向量：

```text
caIn = [
  O2_mass_flow
  N2_mass_flow
  H2O_vapor_mass_flow
  H2O_liquid_mass_flow
  stack_in_T_C
  stack_in_p_abs_kPa
  wetFlag
]
```

注意：与 `BenchAirSupplySimplified` 输出不同，`caIn(1:4)` 是质量流量，不是质量分数。该模块不改变 O2/N2 干气质量流量，只可能把一部分水蒸气转成液态水。

### 连线关系

- 输入 `mix` 来自 `EGRMixerSimplified.mix`。
- 输入 `physical` 来自 `PhysicalParam`。
- 输入 `boundary` 来自 `CaseBoundaryParam`。
- 输出 `caIn` 同时连接到：
  - `PEMFCStackCore.caIn`
  - `SystemSummarySimplified.caIn`
  - `stack_in_node` 工作区记录块

### 核查结论

- 当前接口与脚本参数定义一致：阴极入口温度和压力来自 `stack_in_T_C`、`stack_in_p_kPa + p_amb_kPa`。
- 当前模块符合“入口质量流量来自实验入堆流量，入口温压采用实测入堆边界”的建模口径。
- 已取消饱和水蒸气压函数内部的温度静默夹取，改为入口断言检查；超出公式适用范围时应报错。
- 保留 `min(pSat, 0.98*p)`，因为这是水蒸气分压不能超过总压的物理限制，不是对输入数据的静默修正。
- 模块只做入口状态整理和超饱和水汽冷凝，不承担电堆反应、出口流量或压降计算。

### 验证记录

使用 cEGR 与 no-EGR 各一个代表工况做 1 秒冒烟仿真，`stack_in_node` 输出均为 7x1 向量：

```text
mode=egr, case=cegr0608_002
sum(caIn(1:4)) = 0.00121111 kg/s
target = 0.00121111 kg/s
T_C = 51.72
p_kPa = 141.085
wetFlag = 0

mode=noegr, case=initial_noegr_01
sum(caIn(1:4)) = 0.001071035 kg/s
target = 0.001071035 kg/s
T_C = 58.9
p_kPa = 141.325
wetFlag = 0
```

## PEMFCStackCore

### 模块定位

`PEMFCStackCore` 是当前简化台架模型的核心模块。它接收整理后的阴极入口、上一时刻电堆状态、物性参数、电堆模型参数、单工况边界和冷却曲线，计算：

- 阴极 O2/N2/H2O 与阳极 H2/H2O 的低维气体库存更新。
- O2/H2 反应消耗和电化学生水。
- 膜水拖曳、扩散、跨膜净迁移及一阶松弛。
- 阴极/阳极出口等效流量。
- Nernst 电压、活化损失、欧姆损失和单片电压；当前电流范围暂不启用浓差极化。
- 电堆 lumped 热平衡。
- 出口气体向量、下一步状态和诊断 `summary`。

### 输入

- `caIn`：来自 `BenchInletConditioner` 的阴极入口向量，`caIn(1:4)` 是 O2/N2/H2O 气相/液相质量流量，`caIn(5:6)` 是入堆温度和绝压。
- `x`：来自 `StackStateDelay` 的上一时刻状态：

```text
x = [
  mO2_ca
  mN2_ca
  mH2O_ca
  mH2_an
  mH2O_an
  T_stack_C
  mMem_kg_s
]
```

- `physical`：来自 `PhysicalParam_simplified`：
  - `physical(1:8) = [R; F; M_O2; M_N2; M_H2O; M_H2; p_amb; T_amb]`
- `stackModel`：来自 `StackModelParam_simplified`：
  - `1:4`：电池片数、有效面积、阴极体积、阳极体积。
  - `5:6`：阴极/阳极出口等效导纳。
  - `7:9`：热容、冷却换热、环境换热。
  - `10:19`：Nernst、电压拟合参数、热中性电压和膜水松弛时间：
    - `10`：`E_nernst_ref_V`
    - `11`：`E_nernst_temp_coeff_V_K`
    - `12`：`ASR0_ohm_cm2`
    - `13`：`j0_c_A_cm2`
    - `14`：`conc_loss_c`
    - `15`：`iL_A_cm2`
    - `16`：`sigma_pem_correction`
    - `17`：`alpha_O2`
    - `18`：`thermoneutralVoltage_V`
    - `19`：`tau_mem_s`
  - `PEMFCStackCore` 内部固定书籍参考常数，不再通过 `StackModelParam` 输入：
    - `alpha_H2 = 0.5`
    - `I0_a = 0.1 A/cm2`
    - `I_leak = 0.01 A/cm2`
    - `delta_PEM = 0.0025 cm`
    - `rho_PEM = 1980 kg/m3`
    - `EW = 1.1 kg/mol`
- `boundary`：来自 `CaseBoundaryParam_simplified`，本模块实际读取：
  - `boundary(1)`：电堆电流。
  - `boundary(8)`：阴极出口背压绝压。
  - `boundary(12:14)`：阳极入口总流量、H2 质量分数、水蒸气质量分数。
  - `boundary(15)`：阳极出口背压绝压。
  - `boundary(16:17)`：冷却入口温度、冷却流量。
- `cooling`：冷却流量-换热系数曲线参数。
- `dt`：模型离散步长。

### 计算内容

1. 解包所有物理常数、模型参数和边界条件。
2. 对物性、结构参数、边界压力、阳极入口质量分数、状态非负性、入口向量非负性、温度范围和 `dt` 做硬校验。
3. 根据 `cooling` 曲线决定有效冷却换热系数 `hCoolEff`。
4. 由上一时刻库存和温度计算阴极/阳极当前分压。
5. 从 `caIn` 读取阴极入口 O2/N2/H2O 质量流量，计算 O2 计量比 `lambdaO2`。
6. 按电流计算 O2/H2 反应消耗和电化学生水。
7. 先施加入堆流量、反应消耗和产水，形成不含跨膜水通量的 `pre` 基准库存。
8. 基于同一时间步的 `pre` 水库存计算阴极/阳极水活度、膜含水量、拖曳通量、扩散通量和跨膜净水通量。由于 `mMem` 会反过来改变 `pre` 水库存，代码用固定 3 次迭代求显式近似自洽解。
9. 对膜水通量做库存约束，避免某一侧水库存被单步抽成负值。
10. 更新反应、入堆和膜水迁移统一作用后的预出口气体库存。
11. 根据内部压力和出口背压计算阴极/阳极等效出口流量。
12. 计算出口流量作用后的下一步气体库存。
13. 计算 Nernst 电压、活化损失、欧姆损失和单片电压；`etaCon` 当前置零并作为诊断项输出。
14. 计算电功率、反应热、冷却散热、环境散热、气体显热和下一步电堆平均温度。当前采用方案 B：`T_stack` 表示阴极入口/出口平均温度，阴极出口温度按 `T_ca_out = 2*T_stack - T_ca_in` 估算。
15. 按饱和水蒸气库存裁剪下一步水蒸气，裁剪量作为冷凝诊断量。
16. 生成 `xNext`、`caOut`、`anOut` 和 `summary`。

### 输出

- `caOut`：7x1 阴极出口向量：

```text
caOut = [
  O2_out_mass_flow
  N2_out_mass_flow
  H2O_vapor_out_mass_flow
  H2O_liquid_out_mass_flow
  2*T_stack_C - T_ca_in_C
  cathode_back_pressure_abs_kPa
  wetFlag
]
```

- `anOut`：7x1 阳极出口向量：

```text
anOut = [
  H2_out_mass_flow
  0
  H2O_vapor_out_mass_flow
  0
  T_stack_C
  anode_back_pressure_abs_kPa
  wetFlag
]
```

- `xNext`：下一步状态，反馈到 `StackStateDelay`。
- `summary`：67 项诊断量，主要索引如下：
  - `2`：单片电压 `V_cell`。
  - `4:7`：O2、阴极总压、H2、阳极总压诊断。
  - `8:11`：膜含水量、电堆温度、阴极 O2 摩尔分数、阴极 RH。
  - `16:17`：阴极/阳极出口质量流量。
  - `19:21`：入口 O2 分压、入口 O2 摩尔分数、入口 RH。
  - `31`：最大气体守恒残差。
  - `36:39`：Nernst 电压、活化损失、欧姆损失、浓差损失。
  - `40:41`：实际 O2 计量比、阴极入口质量流量。
  - `49:67`：膜水、冷凝和膜通量诊断。

### 连线关系

- 输入 `caIn` 来自 `BenchInletConditioner.caIn`。
- 输入 `x` 来自 `StackStateDelay.y1`。
- 输入 `physical` 来自 `PhysicalParam`。
- 输入 `stackModel` 来自 `StackModelParam`。
- 输入 `boundary` 来自 `CaseBoundaryParam`。
- 输入 `cooling` 来自 `CoolingCurveParam`。
- 输入 `dt` 来自 `dt_s`。
- 输出 `caOut` 同时连接到：
  - `EGRSplitterSimplified.stackOut`
  - `stack_ca_out_node` 工作区记录块
- 输出 `xNext` 同时连接到：
  - `StackStateDelay.u1`
  - `state_vector` 工作区记录块
- 输出 `summary` 连接到 `SystemSummarySimplified.stackSummary`。

### 静默截断审查

已改为硬校验的内容：

- 物性参数、摩尔质量、体积、热容、内部固定膜厚、`tauMem`、`dt` 必须为正。
- 出口背压、阳极入口流量、阳极组分分数、冷却边界、入口 `caIn`、状态 `x` 必须在物理可接受范围内。
- 饱和水蒸气压公式不再对温度做 `min/max` 静默夹取，温度超出 `-40..120 degC` 时直接断言失败。
- 阳极入口 H2/H2O 质量分数不再用 `min(max(...))` 静默夹取。

保留且有物理/数值意义的限幅：

- 气体库存不能为负，反应/出口后的库存用非负下限保护；对应守恒影响进入 `summary(23:31)`。
- 膜水通量不能从阴极或阳极抽走超过当前可用水库存。
- 出口阀模型不允许负压差反向流，单步最多排出 95% 当前气体库存。
- 水蒸气库存不能超过当前温度和体积下的饱和库存，超过部分作为冷凝诊断。
- 电压公式中的 `log` 输入下限用于避免无电流或零交换电流附近的数值奇异；当前暂不计算浓差极化，因此不再对 `iLim > j + jLeak` 做运行断言。

### 核查结论

- 当前 `caOut(6)` 和 `anOut(6)` 输出的是出口背压边界，不是内部库存压力；内部阴极/阳极压力只作为 `summary` 诊断输出。这与当前压力链口径一致。
- 核心电压结构当前为 `V = E - etaAct - etaOhm`，损失项已从原 `theta1~theta10` 半经验形式改为较少参数的机理化形式：
  - `etaAct = R*T/(2*alpha_H2*F) * ln((j+j_leak)/I0_a) + R*T/(4*alpha_O2*F) * ln((j+j_leak)/I0_c)`
  - `etaOhm = j * (delta_PEM/sigma_PEM + ASR0)`，其中 `sigma_PEM = sigma_pem_correction*(0.005193*lambda-0.00326)*exp(1268*(1/303.15-1/T))`
  - `etaCon = 0`；`conc_loss_c` 和 `iL_A_cm2` 暂保留为接口占位，等更高电流数据出现明显传质拐点后再启用。
- 膜厚已按书籍参考值从 `50 um` 改为 `25 um`，直接在 `PEMFCStackCore` 中固定为 `delta_PEM = 0.0025 cm`。
- 本轮把 `ASR0`、`I0_c`、`sigma_pem_correction`、`alpha_O2` 作为电压拟合参数；`lambda` 来自电堆内部膜水含量计算，不作为外部拟合输入。
- 本轮改动是电压公式口径和参数占位更新；尚未把新参数作为最终标定结果。
- 代码内注释已从 `% 1.`、`% 2.` 这类编号标题改为语义标题；未使用的当前库存总压变量 `pCa` 已删除。核心计算使用的是出口前压力 `pCaPre` 和下一步诊断压力 `pCaDiag`。
- 旧状态下仅为膜水活度服务的 `pO2`、`pN2`、`pVAn` 当前分压路径已删除；膜水活度改为基于同一步 `pre` 水库存的 `pVPreMem/pVAnPreMem`。这样膜水、电压、出口流量和诊断都沿用同一时间层的 pre 口径。
- 对当前 3 次 Picard 迭代做了收敛性复算：全 29 个工况、120 秒回放、29000 个采样点中，第 3 次到第 4 次迭代的最大变化为 `4.16167e-09 kg/s`，最大相对变化为 `3.97621e-04`，99 分位相对变化为 `5.10683e-08`。当前参数和时间步下，3 次迭代基本收敛。

## EGRSplitterSimplified

### 模块定位

`EGRSplitterSimplified` 位于 `PEMFCStackCore` 阴极出口之后。它按工况给定的 EGR 分流比例，把电堆阴极出口气体分成回流支路和台架外排支路。

### 输入

- `stackOut`：来自 `PEMFCStackCore.caOut` 的 7x1 阴极出口向量，`stackOut(1:4)` 是 O2/N2/H2O 气相/液相质量流量，`stackOut(5:6)` 是出口温度和出口背压。
- `boundary`：来自 `CaseBoundaryParam_simplified`，当前模块实际读取：
  - `boundary(11)`：EGR 分流比例 `r`，由 `egr_fraction_model` 输入。

### 计算内容

1. 检查 `r` 在 `0..0.95` 范围内；该范围已在初始化脚本中硬校验，模块内再次断言，避免静默夹取。
2. 检查 `stackOut(1:4)` 非负，出口温度在水蒸气压公式适用范围内，出口压力为正。
3. 回流支路 `returnBranch(1:4) = r * stackOut(1:4)`。
4. 外排支路 `benchOut(1:4) = (1-r) * stackOut(1:4)`。
5. 两条支路都继承电堆出口温度、压力和气体向量格式。
6. 生成 `splitSummary`，用于系统级守恒和分流诊断。

### 输出

- `returnBranch`：7x1 回流支路向量，进入 `WaterSeparatorSimplified`。
- `benchOut`：7x1 台架外排支路向量，进入 `SystemSummarySimplified` 并记录到工作区。
- `splitSummary`：7x1 分流诊断向量：

```text
splitSummary = [
  r
  m_return_gas
  m_bench_out_gas
  m_stack_out_gas
  m_stack_liq
  m_return_liq
  m_bench_liq
]
```

其中气相质量只统计 `O2 + N2 + H2O_vapor`，液态水单独统计。

### 连线关系

- 输入 `stackOut` 来自 `PEMFCStackCore.caOut`。
- 输入 `boundary` 来自 `CaseBoundaryParam`。
- 输出 `returnBranch` 连接到 `WaterSeparatorSimplified.in`。
- 输出 `benchOut` 同时连接到：
  - `SystemSummarySimplified.benchOut`
  - `bench_out_node` 工作区记录块
- 输出 `splitSummary` 连接到 `SystemSummarySimplified.splitSummary`。

### 核查结论

- 已取消 `r = min(max(boundary(11),0),0.95)` 的静默夹取，改为 `assert(r >= 0 && r <= 0.95)`。
- 模块只负责按比例分配电堆阴极出口质量流量，不改变温度、压力和组分格式。
- `returnBranch` 是分离器上游回流量；分离器下游 `egr_return_node` 可能因冷凝排水而小于 `returnBranch`，这不是分流误差。
- 气相和液态水分流守恒都在 `splitSummary` 中可直接审计。

## WaterSeparatorSimplified：EGR 回流支路水分离器

### 模块定位

`WaterSeparatorSimplified` 位于 `EGRSplitterSimplified` 之后、`EGRFeedbackDelay` 之前。它接收分流出来的 EGR 回流支路，按分离器温度和压力判断水汽是否超过饱和容量，超额水汽冷凝并随液态水排出；剩余气相进入回流延迟块，再进入混合仓。

该模块只处理 EGR 回流支路，不改变主外排支路。

### 输入

- `in`：来自 `EGRSplitterSimplified.returnBranch` 的 7x1 回流支路向量：
  - `in(1)`：O2 气相质量流量。
  - `in(2)`：N2 气相质量流量。
  - `in(3)`：H2O 水蒸气质量流量。
  - `in(4)`：液态水质量流量。
  - `in(5)`：分离器上游温度，来自电堆阴极出口。
  - `in(6)`：分离器上游压力，来自电堆阴极出口。
  - `in(7)`：湿态标志，当前分离器出口固定清零。
- `physical`：来自 `PhysicalParam_simplified`，当前模块实际读取：
  - `physical(3)`：O2 摩尔质量。
  - `physical(4)`：N2 摩尔质量。
  - `physical(5)`：H2O 摩尔质量。
- `boundary`：来自 `CaseBoundaryParam_simplified`，当前模块实际读取：
  - `boundary(9)`：分离器/EGR 回流出口温度 `separator_T_C`。
  - `boundary(10)`：分离器/EGR 回流出口绝对压力 `separator_p_abs_kPa`。

### 计算内容

1. 检查摩尔质量为正、入口各质量流量非负、入口温压和分离器温压在可计算范围内。
2. 将出口气相温度、压力覆盖为 `boundary(9)` 和 `boundary(10)`。这是台架回流管路/水分离器测得或设定的回流气体条件。
3. 计算回流支路干气摩尔流量和水蒸气摩尔流量。
4. 如果没有气相回流，模块不再人为加入微小干气，直接输出零气相、零相对湿度；可能存在的入口液态水直接进入排水。
5. 如果存在气相回流，则要求干气摩尔流量大于 0。对当前阴极 EGR 来说，只有水蒸气而没有 O2/N2 的回流不符合该模型的气体定义，应直接暴露为输入错误。
6. 用 Antoine 形式的饱和蒸气压公式计算分离器温度下的饱和压力，并限制 `ps <= 0.98*p`，避免饱和压力接近或超过总压时出现不可定义的湿空气容量。
7. 由干气摩尔流量、总压和饱和压力计算可携带的饱和水蒸气质量流量 `mSat`。
8. 若入口水蒸气超过 `mSat`，超额部分冷凝；若未超过，则冷凝量为 0。
9. 出口 `gasOut(4)` 和 `gasOut(7)` 清零，表示液态水已排出，回流到混合仓的是气相支路。
10. 生成 `sepSummary`，用于系统级水分离诊断和质量守恒检查。

这里保留的 `max(gasOut(3)-mSat,0)` 是物理意义上的“冷凝量不能为负”，不是对异常输入的静默修正。异常输入仍由前置 `assert` 暴露。

### 输出

- `gasOut`：7x1 分离后 EGR 气相回流向量，进入 `EGRFeedbackDelay`。
- `sepSummary`：7x1 水分离诊断向量：

```text
sepSummary = [
  drain
  condensed
  inlet_liquid
  outlet_RH
  outlet_T_C
  outlet_p_abs_kPa
  outlet_gas_mass_flow
]
```

其中 `outlet_gas_mass_flow = sum(gasOut(1:3))`，只统计 O2/N2/H2O 水蒸气；液态水进入 `drain`。

### 连线关系

- 输入 `in` 来自 `EGRSplitterSimplified.returnBranch`。
- 输入 `physical` 来自 `PhysicalParam`。
- 输入 `boundary` 来自 `CaseBoundaryParam`。
- 输出 `gasOut` 同时连接到：
  - `EGRFeedbackDelay.u1`，下一步进入 `EGRMixerSimplified.egr`
  - `SystemSummarySimplified.egrReturn`
  - `egr_return_node` 工作区记录块
- 输出 `sepSummary` 连接到 `SystemSummarySimplified.sepSummary`。

### 核查结论

- 已取消原代码中用 `max(...,1e-12)` 人为制造微小干气/总摩尔流量的做法。no-EGR 工况现在保持真实零回流。
- 已取消饱和蒸气压函数内部对温度的静默夹取，改为入口断言 `-40..120 degC`。
- 模块的质量关系清晰：
  - `出口气相 + 冷凝水 = 入口气相`
  - `排水 = 冷凝水 + 入口液态水`
  - `出口气相 + 排水 = 入口气相 + 入口液态水`
- 分离器出口固定为气相回流，`gasOut(4)=0`，`gasOut(7)=0`。

## SystemSummarySimplified：系统级诊断向量汇总

### 模块定位

`SystemSummarySimplified` 位于模型最末端。它不参与气体混合、入口整理、电堆计算、分流或水分离，只负责把核心模块输出的诊断量按固定顺序拼接成 `summary_vector`，供标定脚本和审计脚本读取。

### 输入

- `stackSummary`：来自 `PEMFCStackCore.summary`，长度为 67，是电堆核心诊断向量。
- `caIn`：来自 `BenchInletConditioner.caIn`，用于计算系统级阴极入口氧摩尔分数和入口气相质量流量。

### 核查结论

- 变量名与块名一致，脚本读取路径清晰。
- `Structure With Time` 与现有脚本中的 `out.summary_vector.signals.values` 读取方式一致。
- `MaxDataPoints=1000` 保留末段数据。对当前“稳态性能为主”的研究目标，脚本取最后一个保存点 `end` 可以得到仿真结束时刻结果，同时避免无必要保存完整 120 秒轨迹。
- 如果后续要分析启动瞬态或中间过程，再临时把 `MaxDataPoints` 改为 `inf` 或增大上限；当前核心基础模型不需要这样做。

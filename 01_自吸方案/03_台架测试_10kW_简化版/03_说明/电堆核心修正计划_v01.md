# 电堆核心修正计划 v01

## 目的

本文件用于固定当前简化版台架 CEGR 模型的第一轮修正计划，避免后续修改过程中遗忘已确认的物理口径、求解口径和验收顺序。

当前修正对象为：

- `01_模型/CEGR_TestBench_10kW_SimplifiedEGR_v01.slx`
- `02_脚本/init_testbench_10kw_simplified_egr.m`
- 后续相关标定、审计和可视化脚本

第一轮目标不是追求 CEGR 电压误差最小，而是先修正电堆核心的边界、源项、求解和诊断，使模型具备可信的后续标定基础。

## 已确认问题

1. 当前电堆核心虽然包含五个气体质量状态和一个温度状态，但状态更新在 MATLAB Function 内部用显式 `dt` 手写完成，再通过 `UnitDelay` 回灌，不是严格的连续 ODE 积分。
2. 模型固定步长为 `0.1 s`，但 `StackStateDelay` 和 `EGRFeedbackDelay` 当前采样时间为 `1 s`，与 `StackParam(35)=dt_s=0.1` 不一致。
3. CEGR 回流通过延迟块打断代数环，本质是显式迭代，不是严格稳态回流闭合求解。
4. 旧无 EGR 台架数据有阴极入口压力和出口压力，但当前初始化曾用入口压力减固定值构造出口边界，不能反映高电流密度下的大阴极压差。
5. 新整理数据中 13 个原始无 EGR 点的阴极压差从 `7.5 kPa` 增至 `55.3 kPa`，必须作为关键边界和验证量保留。
6. 当前膜水反扩散使用气相水蒸气浓度，需改为膜水/界面等效浓度差口径。
7. 当前电渗拖曳项疑似未按电堆总片数放大，需检查并修正为堆级总水通量。
8. 当前液态水只做饱和裁剪，没有液水状态；第一轮可接受，但裁剪时序必须修正。
9. 当前水蒸气饱和裁剪发生在温度更新前，会在升温工况下过度冷凝。
10. 当前 EGR 电压经验损失项 `egr_loss_k_V`、`egr_loss_rh_V` 直接进入浓差损失，容易掩盖真实的压力、湿度、膜水和组分问题。

## 已确认修正原则

### 1. 修正优先级

第一轮优先级为：

1. 求解结构和状态回灌步长
2. 数据入口和压力边界
3. 压力-流量诊断
4. 膜水源项
5. 饱和裁剪时序
6. 电压经验损失冻结
7. 分层审计输出

先修边界、守恒和源项，再看电压拟合。

### 2. 数据入口

后续初始化应优先使用已整理的合并工况表：

- `00_输入参数/实验数据/combined_noegr_cegr_fit_points.csv`

该表包含：

- 原始 13 个无 EGR 台架稳态点
- 0608 CEGR 工况点
- 阴极入口压力 `stack_in_p_kPa`
- 阴极出口压力 `stack_out_p_kPa`
- 阴极压差 `cathode_dp_kPa`
- 实测总入堆流量 `stack_in_flow_meter_SLPM`
- EGR 比例 `egr_fraction_model`

`bench_supply_flow_SLPM` 只作为新鲜空气边界流量进入混合器，不用它反算 CEGR 后的有效氧计量比。

### 3. 压力-流量口径

第一轮不让 `K_ca_in` 每个点单独反算并参与拟合。

短期口径：

- 实测总入堆流量作为入口质量流量边界。
- 阴极出口压力使用数据中的 `stack_out_p_kPa`。
- `pcBack = stack_out_p_kPa + p_amb`。
- 保留 `stack_in_p_kPa`、`stack_out_p_kPa`、`cathode_dp_kPa` 做压力诊断。
- 先重点理顺出口压力、内部库存压力、出堆流量和压差诊断。

后续再决定是否恢复入口压力-流量预测。

### 4. 膜水传输口径

第一轮采用准静态膜水模型，不新增膜水动态 ODE。

统一方向：

- `mMem > 0` 表示水从阳极到阴极。

需修正：

- 电渗拖曳应检查并改为堆级总通量。
- 反扩散使用等效膜水浓度差，不直接使用气相水蒸气浓度。
- 初版建议用：
  - `lambda_ca = lambda_eq(a_ca)`
  - `lambda_an = lambda_eq(a_an)`
  - `lambda_m = 0.5*(lambda_ca + lambda_an)`
  - `C_mem_ca - C_mem_an = rho_mem/EW * (lambda_ca - lambda_an)`
- `lambda_m` 必须影响欧姆损失。
- 输出诊断：`lambda_ca`、`lambda_an`、`lambda_m`、`N_drag`、`N_diff`、`mMem`、`etaOhm`。

### 5. 饱和裁剪口径

第一轮不新增液态水动态状态，但必须修正裁剪时序。

当前错误口径：

- 质量更新后立即用旧温度 `T` 裁剪水蒸气。
- 随后才更新温度 `Tn`。

修正口径：

1. 先计算反应、膜水、进出流后的 provisional 质量。
2. 先计算 provisional 温度 `T_next_pre`。
3. 用 `T_next_pre` 计算饱和蒸汽压。
4. 再执行阴极/阳极水蒸气饱和裁剪。
5. 裁剪后重新计算最终分压、RH、诊断输出。

最大水蒸气库存建议直接用控制体体积计算：

```text
m_v_sat = p_sat(T) * V * M_H2O / (R*T)
```

阴极用 `V_ca`，阳极用 `V_an`。

冷凝潜热第一轮可暂不进入能量方程，但需作为第二轮补强项记录。

### 6. 电压拟合口径

第一轮冻结 EGR 经验电压损失：

- `egr_loss_k_V = 0`
- `egr_loss_rh_V = 0`

CEGR 工况的电压影响优先通过以下链路体现：

```text
回流气组分/湿度 -> 混合气 O2/N2/H2O -> pO2/RH/T -> lambda_m -> etaOhm/etaCon -> V_cell
```

第一轮不直接用 EGR 比例扣电压。若修正后仍有系统性残差，再定义未闭合损失项，但不得只以 EGR 比例作为输入。

## 分层验收标准

### 1. 数据边界回放

29 个工况的关键输入必须与 CSV 一致：

- 电流
- 总入堆流量
- 阴极入口压力
- 阴极出口压力
- 阴极压差
- EGR 比例
- 单片电压实验值

### 2. 压力-流量验收

13 个原始无 EGR 点必须能看到阴极压差随电流升高的趋势。

第一轮不要求完美拟合，但不能再被固定 `5 kPa` 假设覆盖。

### 3. 守恒残差验收

O2、N2、H2O_ca、H2、H2O_an 残差应可解释。

膜水项在阴极和阳极必须等量反号。

### 4. 膜水诊断验收

EGR 湿度升高时，应能从诊断中看到：

```text
RH / 水活度 -> lambda_m -> etaOhm
```

### 5. 饱和/冷凝诊断验收

裁剪必须发生在温度更新后。

诊断应包含：

- 裁剪前水蒸气质量
- 裁剪后水蒸气质量
- `p_sat(T_next)`
- RH
- 冷凝量
- 饱和风险 flag

### 6. 电压验收

最后再看：

- 原始 13 点无 EGR 电压误差
- 0608 无 EGR 点电压误差
- 0608 CEGR 点趋势误差
- 分项 `E / etaAct / etaOhm / etaCon`

第一轮不以 CEGR 电压 RMSE 最小作为唯一目标。

## 暂不做事项

第一轮暂不做：

- 膜水动态 ODE
- 阴极液态水库存状态
- GDL/CL 局部传质模型
- 可压缩节流模型
- EGR 经验电压损失重新拟合
- 全局黑箱拟合

这些内容在第一轮审计通过后再决定是否进入第二轮。

```text
J_drag = n_drag * I / (F*A)
```

```text
mMemMaxPos = available_anode_vapor / dt
mMemMaxNeg = -available_cathode_vapor / dt
mMem = clip(mMem_raw, mMemMaxNeg, mMemMaxPos)
```

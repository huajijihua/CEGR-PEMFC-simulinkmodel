## 2. 当前顶层拓扑

当前 `stage1_pressurefix` 为 compact audit-derived 结构：

```text
EnvironmentFreshAir
-> EGRMixer
-> Compressor
-> Intercooler
-> HumidifierDryWetLumped
-> PEMFCStackCore
-> EGRValve
-> BackPressureValveWithOutletManifold
-> ambient
```

两条离散反馈：

- `EGRFeedbackDelay`：上一拍 EGR 回流节点反馈到 `EGRMixer`。
- `WetFeedbackDelay`：上一拍电堆阴极出口节点反馈到 `HumidifierDryWetLumped` 的湿侧入口。

该模型没有独立的 `TailGasManifold`、`Separator`、`EGRReturnPipe` 动态模块；相应阻力、分流和边界作用被集总进相邻模块。

## 3. 气路节点接口

阴极和尾气主流动节点为 7 维列向量：

```text
[m_O2_kg_s
 m_N2_kg_s
 m_H2O_v_kg_s
 m_H2O_l_kg_s
 T_C
 p_kPa
 liquid_present]
```

阳极出口节点也复用同类 7 维接口，但当前主要作为诊断和一致性输出。

接口约束：

- 节点压力 `p_kPa` 表示该节点传递给下游模块的边界压力。
- 电堆内部库存压力不通过节点压力向下游传递。
- `RH`、`xO2`、`pO2`、残差、压降等为诊断量，由 summary 或验证脚本派生。

## 4. 电堆压力语义

### 4.1 堆内库存压力

`PEMFCStackCore` 使用阴/阳极流道几何体积计算堆内库存压力：

```text
V_ca_m3 = 2.0e-4
V_an_m3 = 1.5e-4

pCa = pO2 + pN2 + pH2O_v
pAn = pH2 + pH2O_v
```

这些压力进入 `summary_vector` 中的：

```text
pCa_kPa
pAn_kPa
```

它们只表示电堆内部代表压力，用于压力顺序、氧分压、电压和物理一致性诊断。

### 4.2 出口压力边界

二周目压力修正后，出口节点压力已改为边界压力：

```text
caOut(6) = pcBack
anOut(6) = panBack
```

含义：

- `caOut(6)` 是阴极出口/背压边界压力，不是堆内 `pCa_kPa`。
- `anOut(6)` 是阳极出口/背压边界压力，不是堆内 `pAn_kPa`。
- 下游湿侧加湿器、EGR 阀和尾排模块不得把堆内库存压力当成出口边界压力读取。

### 4.3 出口流量驱动

出口流量仍由堆内压力与出口边界压力差驱动：

```text
mCaOutRaw = K_ca_out * max(pCaPre - pcBack, 0) / limiter
mAnOutRaw = K_an_out * max(pAnPre - panBack, 0) / limiter
```

因此当前压力链主逻辑是：

```text
入口压力边界 -> 堆内库存压力 -> 出口背压边界
```

验收约束为：

```text
p_ca_in_sim_kPa > p_stack_internal_kPa > p_ca_out_boundary_kPa
```

其中：

- `p_ca_in_sim_kPa = humidifier_dry_node(6)`
- `p_stack_internal_kPa = summary.pCa_kPa`
- `p_ca_out_boundary_kPa = P.p_cathode_back_kPa`

## 6. 热侧与湿侧边界

当前二周目不再复刻一周目的完整三阶段流程，而是拆成：

```text
压力修正 -> 热侧 Stage A -> 加湿器优先湿侧边界 -> 必要时再复核电压
```

`init_vehicle_10kw_gzs60_v3.m` 当前会读取：

```text
00_输入参数/标定参数/pressurefix_stage1_boundary_params.csv
00_输入参数/标定参数/thermal_stageA_params.csv
00_输入参数/标定参数/thermal_stageA_cooling_flow_curve.csv
00_输入参数/标定参数/humidity_stageB_params.csv
00_输入参数/电堆物理模型/stack_voltage_book_theta_params.csv
00_输入参数/标定参数/stage2_thermal_humidity_calibrated_params.csv
```

当前热侧 Stage A 的冻结口径：

- `C_stack_J_K` 冻结；
- `hum_heat_eff` 在 Stage A 冻结；
- 冷却增强通过 `coolant_flow_L_min -> h_cool_eff` 等效曲线进入；
- `T_cool_C` 当前取台架冷却液入口温度，用于形成更可辨识的换热驱动力；
- 台架冷却带热量按水当量近似反算，仅作为热侧辅助约束。

因此当前工作基线已经不再使用旧的 `current_A -> h_cool_curve_W_K` 语义。

当前湿侧加湿器优先口径：

- 先独立复核 `HumidifierDryWetLumped` 的四口出口湿态、压降、传水量和传热趋势；
- 电堆阴极入口湿度由加湿器干侧出口决定；
- 台架 `RH_ca_in`、`pH2O_caIn`、`xH2O_caIn` 只作对比/校核，不作为车载膜加湿器出口硬拟合目标；
- `humidity_stageB_params.csv` 当前保存加湿器优先参数，旧 Stage B 第一轮作为历史对照；
- 不主动改动堆内膜水参数、压力链参数或 `C_stack_J_K`。

`HumidifierDryWetLumped` 当前传水源项为低维 epsilon-NTU 水容量模型：

```text
NTUgain = hum_NTU_ref * (m_ref / m_dry)^hum_flow_exp
epsilon = 1 - exp(-NTUgain)
m_transfer = epsilon * min(dry_need, wet_available)
```

其中 `dry_need` 为干侧达到当前出口温度饱和湿度的可吸收水量，`wet_available` 为湿侧可供给水蒸气量。

当前传热源项为系统级集总有效度模型：

```text
epsilon_H = 1 - exp(-UA / C_min)
dT_dry = epsilon_H * (T_wet_in - T_dry_in) * C_min / (m_dry * cp_gas)
T_dry_out = T_dry_in + dT_dry
T_wet_out = T_wet_in - dT_dry
```

该写法用于研究膜加湿器对系统入口水热边界的趋势影响，依据为系统级 PEMFC 建模中常用的集总换热/换湿口径。它不解析膜内温度场、壁面热容或沿程分布，也不声明 GZS60 四端口已经完成强定量标定。当前验收重点是干侧增湿方向、湿侧失水方向、传水上限、压降量级和与供应商规格点的一致性。

## 7. 热侧方程补充

`PEMFCStackCore` 当前热侧主项为：

```text
Q_gen  = I * N * (Vtn - V)
Q_cool = h_cool_eff(flow_coolant) * (T_stack - T_cool_in)
Q_amb  = h_amb * (T_stack - T_amb)
Q_gas  = m_gas_out * cp_gas * max(T_stack - T_ca_in, 0)

C_stack * dT_stack/dt = Q_gen - Q_cool - Q_amb - Q_gas
```

其中：

- `flow_coolant = coolant_flow_L_min`
- `h_cool_eff` 由 `thermal_stageA_cooling_flow_curve.csv` 给出
- `T_cool_in` 当前取台架 `coolant_inlet_temp_C`

热侧 Stage A 用于先把稳态热平衡立住，不在本阶段引入完整冷却液动态回路、泵阀控制或散热器模块。

## 8. 电堆液水与电压模型边界

当前电堆气体库存状态只保留阴极 O2/N2/H2O_v、阳极 H2/H2O_v 和电堆温度。若水蒸气分压超过饱和蒸气压，模型对水蒸气质量进行饱和裁剪，并把液化量作为诊断量输出。该液化量当前等效为瞬时排出或未显式跟踪的液水去向，不作为独立液水库存、排水通道或液水滞留状态。

当前电压方程沿用书籍系统模型形式和已有循环曲线标定结果，不在本阶段更改方程结构。氧分压和入口状态用于系统趋势分析；若后续获得多条循环/EGR 实测曲线，再考虑重新辨识氧传质或浓差损失相关项。
